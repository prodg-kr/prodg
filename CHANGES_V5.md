# pronews.jp v5 - DAILY_LIMIT 제거

## ✅ 변경사항

### DAILY_LIMIT 제거 - 모든 RSS 기사 처리

**변경 전 (v4):**
```python
DAILY_LIMIT = 10  # 하루 최대 10개

# ...
return all_articles[:DAILY_LIMIT]  # 10개만
```

**변경 후 (v5):**
```python
# DAILY_LIMIT 제거

# ...
return all_articles  # 모든 기사 처리
```

## 🎯 작동 방식

### 매일 실행 시:

1. **RSS 피드 전체 확인**
   - pronews.jp의 모든 RSS 항목 읽기 (보통 15-20개)

2. **이미 게시된 기사 건너뛰기**
   - `posted_articles.json`에 있는 기사는 자동 제외

3. **새 기사만 번역**
   - 중복 체크 후 새 기사만 처리

4. **모두 게시**
   - 제한 없이 모든 새 기사 게시

## 📊 예시

### 시나리오 1: 첫 실행
```
RSS 피드: 20개 기사
posted_articles.json: 0개 (비어있음)

→ 20개 모두 번역 및 게시
→ 소요 시간: 약 20-30분
```

### 시나리오 2: 다음날 오전 7시
```
RSS 피드: 20개 기사
- 어제 기사 15개 (이미 게시됨)
- 새 기사 5개

→ 5개만 번역 및 게시
→ 소요 시간: 약 5-10분
```

### 시나리오 3: 뉴스가 많은 날
```
RSS 피드: 25개 기사
- 어제 기사 10개 (이미 게시됨)
- 새 기사 15개

→ 15개 모두 번역 및 게시
→ 소요 시간: 약 20-30분
```

## ✅ 장점

### 1. 기사 누락 없음
- RSS에 있는 모든 새 기사 자동 게시
- 하루에 20개가 나와도 모두 처리

### 2. 유연성
- 뉴스가 많은 날: 모두 게시
- 뉴스가 적은 날: 적게 게시
- 자동으로 조절됨

### 3. 간편한 관리
- WordPress 관리자에서 직접 삭제 가능
- 불필요한 기사는 나중에 정리

### 4. 중복 방지
- 자동으로 이미 게시된 기사는 건너뜀
- 수동으로 삭제 걱정 없음

## 🔧 제한을 다시 추가하고 싶다면?

필요시 제한 추가 가능:

```python
# 설정 섹션에 추가
MAX_ARTICLES_PER_RUN = 20  # 원하는 개수

# fetch_rss_feed 함수에서
return all_articles[:MAX_ARTICLES_PER_RUN]
```
